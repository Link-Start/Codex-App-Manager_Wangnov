# .impeccable

设计上下文（Design Context），用于指导本项目所有前端/设计决策。

## Design Context

### Users
使用（或准备安装）官方 Codex 桌面应用的开发者。他们偶尔、短暂地打开这个 menubar/popover 风格的小工具，用来：安装、检查更新、执行更新、切换更新源、卸载，以及（即将到来的）管理 `~/.codex` 配置。使用场景多发生在编码前后，身边常开着深色 IDE。

这个工具会触碰用户已安装的应用，因此**信任**是第一位的；其次是**速度**与**清晰**。跨平台：macOS（主）+ Windows。11 种语言、含 RTL（阿拉伯语）。窗口固定 400×640、无边框、透明、自绘关闭。

### Brand Personality
- 三个词：**温润（crafted/tactile）· 可信（trustworthy）· 精密（quietly premium）**。
- 语气：冷静、自信、精确——绝不喧哗、绝不推销。它像 OpenAI Codex 的官方随身管家。
- 情绪目标：让人**放心**把已装应用交给它管理；操作过程中**安定**；藏一点**克制的愉悦**，让人愿意反复打开。

### Aesthetic Direction
- **自信进化（confident evolution）**：保留现有 Linear/Raycast 开发者工具的基因与品牌资产（紫→蓝 云+终端 logo、环形状态 hero、popover 外壳），但重构视觉层次、排版、配色、表面材质与动效，使其更精致、更有辨识度。不换方向。
- **温润材质感（tactile / warm material）**：真实的深度与分层表面；单一一致光源下物理可信的柔和光影；克制的材质质感（极轻的颗粒/高光，而非到处玻璃拟态）；克制但有惊喜的微动效。
- **双主题同等精雕**：深色默认、浅色为同等一流的对等主题——靠 token 承载"灵魂"，让两者都像被精心打磨，而非简单反色。
- 反面参考（务必避免）：通用 AI 仪表盘、深底霓虹青、紫→蓝渐变文字、到处玻璃拟态、卡片左侧色条、千篇一律的等大卡片网格、图标+标题+正文重复堆叠。

### Constraints
- 技术栈：React 19 + TypeScript + Vite + Tauri v2。设计系统集中在 `src/app/styles.css`（语义类名，组件结构良好）。
- 固定 400×640 无边框透明窗口（`#root` 留 14px 透明 gutter 供圆角+阴影）。
- 必须保留 i18n（11 语言）+ RTL + `prefers-reduced-motion` 支持。
- 品牌色与 logo 保留。离线桌面应用：**不引入网络/打包 web 字体**——用平台系统字体精心调校（mac 圆体 SF Pro Rounded 天然契合"温润"；Windows 用 Segoe UI；CJK 落到 PingFang/YaHei；技术读数用系统等宽 + tabular 数字承载"开发者/Codex"identity）。辨识度来自材质、配色、空间、动效与构图，而非换字体。

### Design Principles
1. **材质有意图（Material with intent）**：每个表面的高度都有理由；光来自单一一致方向；深度用于表达层次，而非装饰。
2. **冷静的自信（Calm confidence）**：克制胜过喧哗；每屏一个焦点（环形状态）；充足呼吸感；强调色稀有且有意义（60-30-10）。
3. **让人安心的动效（Motion that reassures）**：编排式入场与状态转场，让操作显得安全、有生命力；指数缓出、无弹跳；尊重 reduced-motion。
4. **两个对等、一个灵魂（Two peers, one soul）**：深浅皆一等公民；token 承载灵魂，两者都精雕，而非"反色"。
5. **细节即信任（Trust in the details）**：tabular 数字、精确对齐、诚实的进度、任意语言/长度都易读；杜绝任何"AI slop"痕迹。
